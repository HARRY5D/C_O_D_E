SQL AND RELATIONAL ALGEBRA

1. RELATIONAL MODEL
The relational model represents data as a collection of tables (relations).
- Relation (Table): A set of tuples (rows).
- Attribute (Column): Each attribute has a name and a domain (set of allowed values).
- Tuple (Row): A single data record.
- Primary Key: A minimal set of attributes that uniquely identifies each tuple.
- Foreign Key: An attribute (or set of attributes) that references the primary key of another relation.

2. RELATIONAL ALGEBRA
Relational algebra is a procedural query language that consists of a set of operations.

Fundamental Operations:
- Select (σ): σ_condition(R) - Selects tuples satisfying condition.
- Project (π): π_attr-list(R) - Selects specified columns.
- Union (∪): R ∪ S - Combines all tuples from both relations (same schema required).
- Set Difference (-): R - S - Tuples in R but not in S.
- Cartesian Product (×): R × S - All possible combinations of tuples.
- Rename (ρ): ρ_new-name(R) - Renames a relation.

Derived Operations:
- Natural Join (⋈): R ⋈ S - Join on common attributes.
- Theta Join (⋈θ): R ⋈_θ S - Cartesian product followed by selection.
- Division (÷): R ÷ S - Tuples in R that are related to all tuples in S.
- Intersection (∩): R ∩ S - Tuples in both R and S.

3. SQL (STRUCTURED QUERY LANGUAGE)

DDL (Data Definition Language):
CREATE TABLE Employee (
    EmpID INT PRIMARY KEY,
    Name VARCHAR(50) NOT NULL,
    DeptID INT,
    Salary DECIMAL(10,2),
    FOREIGN KEY (DeptID) REFERENCES Department(DeptID)
);

DML (Data Manipulation Language):
INSERT INTO Employee VALUES (101, 'Alice', 10, 75000.00);
UPDATE Employee SET Salary = 80000 WHERE EmpID = 101;
DELETE FROM Employee WHERE EmpID = 101;

DQL (Data Query Language):
SELECT E.Name, D.DeptName, E.Salary
FROM Employee E
INNER JOIN Department D ON E.DeptID = D.DeptID
WHERE E.Salary > 50000
ORDER BY E.Salary DESC;

Aggregate Functions:
SELECT DeptID, COUNT(*) AS NumEmp, AVG(Salary) AS AvgSal
FROM Employee
GROUP BY DeptID
HAVING AVG(Salary) > 60000;

4. GATE PREVIOUS YEAR QUESTIONS

Q1. (GATE 2023) Consider two relations R(A,B) and S(B,C).
Natural join R ⋈ S will have schema:
(A) (A,B)  (B) (B,C)  (C) (A,B,C)  (D) (A,C)
Answer: (C) (A,B,C)
Explanation: Natural join combines on common attribute B, resulting in schema (A,B,C).

Q3. (GATE 2021) Which of the following SQL queries finds employees who earn more than
the average salary of their department?
(A) SELECT * FROM Emp E WHERE Salary > AVG(Salary);
(B) SELECT * FROM Emp E WHERE Salary > (SELECT AVG(Salary) FROM Emp WHERE DeptNo = E.DeptNo);
(C) SELECT * FROM Emp E HAVING Salary > AVG(Salary);
(D) Both (A) and (C)
Answer: (B) - Correlated subquery computes avg salary per department.

5. NORMALIZATION

Normal Forms:
1NF: Each attribute contains only atomic values. No repeating groups.
2NF: In 1NF and every non-prime attribute is fully functionally dependent on the primary key.
     (No partial dependency)
3NF: In 2NF and no transitive dependency (non-prime attribute depends on another non-prime attribute).
BCNF: In 3NF and for every functional dependency X → Y, X is a superkey.

Functional Dependencies:
Armstrong's Axioms:
1. Reflexivity: If Y ⊆ X, then X → Y.
2. Augmentation: If X → Y, then XZ → YZ.
3. Transitivity: If X → Y and Y → Z, then X → Z.

Derived rules:
- Union: If X → Y and X → Z, then X → YZ.
- Decomposition: If X → YZ, then X → Y and X → Z.
- Pseudotransitivity: If X → Y and WY → Z, then WX → Z.

6. TRANSACTIONS AND CONCURRENCY CONTROL

ACID Properties:
- Atomicity: Transaction is treated as a single unit; either fully completed or fully rolled back.
- Consistency: Transaction brings database from one valid state to another.
- Isolation: Transactions execute independently without interference.
- Durability: Once committed, effects persist even if system fails.

Concurrency Problems:
- Lost Update: Two transactions read and update same data; one update is lost.
- Dirty Read: Read uncommitted data from another transaction.
- Non-repeatable Read: Read same data twice within transaction; values differ.
- Phantom Read: Query returns different rows in two executions due to inserts/deletes.

Isolation Levels (SQL standard):
- Read Uncommitted: Allows dirty reads.
- Read Committed: Prevents dirty reads.
- Repeatable Read: Prevents dirty and non-repeatable reads.
- Serializable: Prevents all concurrency problems.
