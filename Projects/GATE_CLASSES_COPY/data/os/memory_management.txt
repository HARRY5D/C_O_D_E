MEMORY MANAGEMENT IN OPERATING SYSTEMS

1. INTRODUCTION
Memory management is the functionality of an OS which handles or manages primary memory.
It monitors each byte in a computer's memory and checks whether it is allocated or free.

2. LOGICAL vs PHYSICAL ADDRESS SPACE
- Logical Address: Address generated by the CPU, also referred to as virtual address.
- Physical Address: Address seen by the memory unit (loaded into MAR).
The set of all logical addresses is the logical address space.
The set of all physical addresses is the physical address space.
The mapping from virtual to physical addresses at runtime is done by the MMU (Memory Management Unit).

3. CONTIGUOUS MEMORY ALLOCATION
Each process is contained in a single contiguous section of memory.
Partition schemes:
- Fixed-size partitions: Memory is divided into fixed-size partitions.
- Variable-size partitions: Memory is divided based on process needs.

Strategies for selecting a free hole:
- First-fit: Allocate the first hole that is big enough.
- Best-fit: Allocate the smallest hole that is big enough. Produces smallest leftover hole.
- Worst-fit: Allocate the largest hole. Produces the largest leftover hole.

4. FRAGMENTATION
- External Fragmentation: Total memory space exists to satisfy a request but is not contiguous.
- Internal Fragmentation: Allocated memory may be slightly larger than requested memory; the difference is internal fragmentation.

5. PAGING
Paging is a memory management scheme that eliminates the need for contiguous allocation of physical memory.
- Physical memory is divided into blocks of fixed size called frames.
- Logical memory is also divided into blocks of the same size called pages.
- Page table maps logical pages to physical frames.

Address Translation:
- Logical address = (page number p, page offset d)
- Physical address = (frame number f, page offset d)
- Frame number obtained from page table using page number as index.

6. GATE PREVIOUS YEAR QUESTIONS

Q1. (GATE 2023) Consider a system with byte-addressable memory, 32-bit logical addresses,
4KB page size, and page table entries of 4 bytes each.
The number of bits required for the page offset is:
(A) 10  (B) 12  (C) 14  (D) 16
Answer: (B) 12
Explanation: Page size = 4KB = 2^12 bytes. So page offset needs 12 bits.

Q2. (GATE 2022) A virtual memory system uses FIFO page replacement policy and has 4 page frames.
The reference string is: 1 2 3 4 1 2 5 1 2 3 4 5
Number of page faults (including compulsory misses) is:
(A) 8  (B) 9  (C) 10  (D) 12
Answer: (C) 10

Q3. (GATE 2021) Belady's anomaly is associated with which page replacement algorithm?
(A) LRU  (B) Optimal  (C) FIFO  (D) MRU
Answer: (C) FIFO
Explanation: Belady's anomaly is a phenomenon where increasing the number of page frames
results in more page faults in FIFO page replacement algorithm.

7. SEGMENTATION
Segmentation is a memory management technique in which the memory is divided into variable-size parts
called segments. A segment is a logical unit such as: main program, procedure, function, stack, arrays.

Segmentation with paging (Intel x86):
- Segment selector → Segment descriptor → Linear address
- Linear address → Page directory → Page table → Physical address

8. VIRTUAL MEMORY
Virtual memory is a technique that allows the execution of processes that are not completely in memory.
Benefits:
- Programs can be larger than physical memory.
- More programs can run simultaneously.
- Less I/O needed to load or swap processes.

Demand Paging: Load pages only as needed.
Page Fault: Occurs when a process accesses a page not in memory.

Page Replacement Algorithms:
1. FIFO: Replace the oldest page.
2. Optimal (OPT): Replace the page that will not be used for the longest time. Not implementable.
3. LRU: Replace the page not used for the longest time in the past.
4. LRU Approximation: Clock algorithm, Second-chance algorithm.

Thrashing: If a process does not have enough pages, the page-fault rate is very high.
This is called thrashing. A process is thrashing if it is spending more time paging than executing.

Working Set Model: Uses parameter Δ (working set window) to approximate locality model.
Working set = set of pages in most recent Δ page references.
